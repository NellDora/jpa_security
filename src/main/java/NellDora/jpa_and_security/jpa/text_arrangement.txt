

객체 연관관계

객체 -> 객체(참조)
객체 <- 객체(참조)

DB의 경우
테이블 <- (외래키 - Join) -> 테이블 (참조)
데이터베이스에서는 외래키를 사용하기 때문에 양방향이 가능하지만 자바 객체에서는 불가능

자바상에서 두 객체간의 내용에서 나오는 것이 주인관계

양방향 매핑을 위한 정리
 - 객체의 두 관계중 하나를 연관관계의 주인으로 지정
 - 연관관계의 주인만 외래키를 관리( 등록, 수정)
 - 주인이 아닌 쪽은 읽기만 가능
 - 주인은 mappedBy 속성 부여하지 않음
 - 주인이 아닌 경우 mappedBy 속성 부여 (주인 지정)  * 주인지정은 외래키가 있는 곳으로 지정

 양방향 관계보다 단방향이 더욱이 안정적임

 실제 설계시 첫 시작은 모두 단방향으로 설계를 할 것

 이후 필요한 경우에만 양방향 설계 추가 (DB 테이블 수정 필요 X / 오직 자바 코드로만 추가)
  -> 필요 할 때만 추가

 양방향은 설계를 위해서가 아닌 실제 나중에 조회할 때 편의를 위해서 설계하는 것
